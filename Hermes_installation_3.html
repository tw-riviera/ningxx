Hermes Agent 安裝完整學習手冊

使用者：ko（零 Coding 基礎）
日期：2026年5月

一、重要電腦概念（白話解釋）

VPS（Virtual Private Server）

你在雲端租的一台「24小時永遠開機的電腦」。你現在的 Hermes 就是跑在這台電腦上。

Docker（容器）

把 Hermes 整個程式包在一個「密封的盒子」裡。優點是安裝乾淨、不容易跟其他程式衝突，也比較安全。

Gateway（閘道 / 電話線）

Hermes 的「電話線」。沒有開 Gateway 的時候，你只能在黑畫面（終端機）跟 Hermes 聊天。開了 Gateway 之後，你就可以用 Telegram 隨時找它聊天。

終端機（黑畫面）

直接用文字指令跟電腦溝通的地方。你現在打的那些指令就是在終端機裡執行的。

Provider（提供者）

Hermes 要用哪一個 AI 模型（Kimi、DeepSeek、Qwen 等）。不同的 Provider 有不同的規則和問題。

API Key

像一組密碼，讓 Hermes 可以合法呼叫 Kimi / DeepSeek 等 AI 服務。

二、你執行過的重要指令 + 詳細解釋

1. 檢查 Gateway 是否在運行（最常用）

ps aux | grep gateway

詳細解釋：

  • ps aux = 列出目前電腦上「所有正在運行的程式」
  • | = 管道符號，把左邊結果傳給右邊繼續處理
  • grep gateway = 只顯示包含 "gateway" 這幾個字的行

→ 這條指令是用來檢查「Hermes 的電話線（Gateway）」有沒有在跑。

2. 重啟 Gateway

HERMES_ALLOW_ROOT_GATEWAY=1 hermes gateway restart

解釋： 強制重新啟動 Telegram 連接。
前面那串 HERMES_ALLOW_ROOT_GATEWAY=1 是因為你在 Docker 容器裡用 root 身份，需要特別允許。

3. 查看 Docker 容器狀態

docker ps

解釋： 列出目前正在運行的 Docker 容器。你可以看到 Hermes 的容器是否在運行。

4. 設定容器自動重啟（非常重要！）

docker update --restart unless-stopped hermes-agent-1hhn-hermes-agent-1

解釋： 告訴 Docker：「除非我手動停止這個容器，否則不管發生什麼事（VPS 重啟、程式崩潰），都要自動重新啟動它。」這是目前讓 Hermes 最穩定的做法。

5. 切換模型 Provider

HERMES_ALLOW_ROOT_GATEWAY=1 hermes config set provider moonshot

解釋： 告訴 Hermes：「以後主要用 Moonshot（Kimi）這個提供者」，而不是有問題的 kimi-coding。

6. 設定具體模型

HERMES_ALLOW_ROOT_GATEWAY=1 hermes config set model kimi-k2.6

指定使用 Kimi 的 k2.6 版本。

7. 背景運行 Gateway

nohup hermes gateway run > /tmp/gateway.log 2>&1 &

解釋：
- nohup：即使你關掉終端機，程式還是繼續跑。
- > /tmp/gateway.log：把輸出記錄到這個檔案。
- 2>&1：錯誤訊息也一起記錄。
- &：放到背景執行。

8. 查看日誌（出問題時最常用）

cat /tmp/gateway.log | tail -30

查看 Gateway 最近的 30 行記錄，幫助找出問題。

9. 進入聊天模式測試

hermes chat

在終端機直接跟 Hermes 聊天（不用 Telegram）。

10. 離開 Docker 容器

exit

從 Hermes 容器裡面回到 VPS 主機。

三、你走過的完整流程（簡要時間線）

  1. 在 Hostinger 購買 VPS 並用 Docker 一鍵部署 Hermes
  2. 設定 Admin 帳號密碼
  3. 執行 hermes setup 進行初始設定
  4. 設定 Telegram Bot Token
  5. 多次遇到 kimi-coding 導致 400/401 錯誤
  6. 多次切換 Provider（kimi-coding → moonshot）
  7. 處理僵屍進程、Token 被替換成 placeholder 的問題
  8. 切換到 DeepSeek（後來因額度問題放棄）
  9. 回頭使用 Kimi 並充值 Tier 1
  10. 設定安全參數（redact_pii 等）
  11. 設定容器自動重啟（unless-stopped）

四、目前最終狀態（2026年5月）

  • 模型：Kimi k2.6（已正確使用 moonshot provider）
  • Telegram Bot：可正常聊天
  • Gateway：背景運行
  • 自動重啟：已設定（unless-stopped）
  • 安全性：已做基本加固

六、重要提醒

  • 每次改設定後，記得重啟 Gateway。
  • 如果 Telegram 突然沒回應，先檢查 ps aux | grep gateway。
  • 想長期穩定，容器自動重啟設定非常重要。
  • 有問題時，先看日誌 cat /tmp/gateway.log | tail -30。
