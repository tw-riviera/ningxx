🤖 1. Hermes 是什麼？

你的「萬能數碼助手」，住在雲端（或你的電腦裡）的超級秘書。

  • 會說話、會思考（接駁 Kimi/Moonshot AI 大腦）
  • 有「手腳」能執行動作——發訊息、整理檔案、搜尋網頁、寫文件、設定自動提醒
  • 多平台連接（Telegram、WhatsApp、Discord、電郵等）
  • 有記憶——記得你是誰、你的偏好、你在做什麼項目

📁 3. Session 檔案系統

Session = 聊天記錄的存檔。每次對話自動保存，讓 Hermes 記得你。

檔案類型 誰產生 用途 能否刪除
.jsonl Gateway 層 逐行原始對話記錄（一句一行） 不能 會遺失對話記憶
.json run_agent.py 引擎備份快照（含 system prompt、工具定義） 暫時可 但會自動再生
sessions.json 系統 目錄索引（記錄目前活躍 session） 不能 系統會迷路
request_dump_*.json 除錯系統 網路請求臨時除錯記錄 可以 安全清理
🔍 展開：.json 與 .jsonl 的詳細對比
比較項目.json.jsonl
格式單一大塊 JSON每行一個獨立 JSON
寫入方式每次覆寫整個檔案追加在最後一行
大小較大（含系統提示、工具清單）較小（只含對話內容）
CLI 模式✅ 有❌ 沒有
Telegram 模式✅ 有✅ 有
Resume 來源不直接使用（備份用）不直接使用（Gateway 記錄用）
實際 Resume 來自SQLite (state.db)
🔑 4. Root 是什麼？

電腦世界裡的「萬能鑰匙持有者」。

👤 普通用戶

像住戶——能進自己房間、用廚房、看電視

👑 Root（超級管理員）

像大廈業主——能進任何房間、拆牆、改電線、甚至拆掉整棟樓

Hermes 現在以 root 身份運行。在單人用戶的容器環境中風險可控，但長遠應改為普通用戶運行。

⚠️ 6. 使用風險

🔴 高：權限過大

Hermes 能讀寫檔案、執行指令。設定錯誤或被他人控制會有危險。

對策：已鎖定 Telegram、開啟手動審批

🟠 中：隱私與資料儲存

對話記錄存在本地，含個人資訊。API 金鑰儲存在設定檔中。

對策：已開啟 PII 遮蔽、清理舊 session

🟡 低：AI 幻覺

AI 有時會「自信地胡說」，給出錯誤建議。

對策：重要決定二次確認

🟢 可控：成本

每次對話消耗 API Token，文字越多越貴。

對策：留意 Moonshot 帳單

🚀 7. Hermes 如何優化生活？

💡 8. 今日學習金句
  • 「Session 是聊天記憶的存檔，讓 AI 記得你是誰」
  • 「.json 是引擎備份，.jsonl 是 Gateway 原始記錄」
  • 「Root 是電腦的萬能鑰匙持有者，能力越大責任越大」
  • 「Resume 對話從 SQLite 載入，不是從 .json」
  • 「刪 .json 只會暫時清空間，對話繼續就會自動再生」
  • 「把 Hermes 當副駕駛，不是自動駕駛——方向盤在你手中」
