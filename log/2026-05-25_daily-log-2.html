Cold War

React SPA deployment — from Kimi Web Swarm to GitHub Pages.

01The Mission

用戶要求：研究 react-gh-pages，部署 cold-war-2（後改為 cold-war-3）React 網站到 GitHub Pages。

這是一個 Vite + React + TypeScript + Tailwind 專案，由 Kimi Web Swarm 生成。主題是電影《寒戰》的機密檔案視覺風格。

02The Deployment

2.1 研究階段

執行動作

  • 讀取 github.com/gitname/react-gh-pages 官方 repo
  • 區分 CRA (create-react-app) vs Vite 兩種部署方式
  • 確認 cold-war-3 是 Vite 專案 → 需要 HashRouter + base: './'
  • 安裝 gh-pages npm package

2.2 部署流程

標準 workflow

  • git clone tw-riviera/cold-war-3
  • 確認 vite.config.ts 有 base: './'
  • 確認 src/main.tsx 使用 HashRouter（非 BrowserRouter）
  • npm run build → 輸出 dist/ 目錄
  • npx gh-pages -d dist → 推送到 gh-pages branch
  • GitHub Pages settings: gh-pages branch / root

2.3 遇到的問題

問題與修復

  • 圖片 404：程式碼用絕對路徑 src="/dossier-xxx.jpg" → 改為相對路徑 src="./dossier-xxx.jpg"
  • 圖片是 placeholder：Kimi 生成的 JPEG 是合法但均勻灰色 → 需替換真實圖片
  • HashRouter 跳轉不捲動：加入 ScrollToTop 組件，每次路由變化 window.scrollTo(0, 0)
  • 第 6 版塊無跳轉：建立 OperativeStatus.tsx 頁面，補上路由和連結
  • 修改了錯誤路徑：cold-war-3-app/ 而非 cold-war-3/ → 永遠先 find 確認

03The Polish

3.1 互動細節

用戶要求 6 個版塊加入「click to access」提示。實現方式：group-hover + opacity-0 → opacity-100 淡入效果。僅對可點擊版塊顯示，第 6 個（OPERATIVE STATUS）純顯示，無提示。

3.2 打字機效果

用戶要求將 scroll-driven typing（scroll 位置控制文字量）改為 typewriter effect（scroll 觸發後自動逐字打出）。修改 ScrollTrigger：scrub: true → once: true + setInterval 30ms/char。

3.3 加入 Hermes_VPS 索引

更新 artwork-table-of-content.html（Projects 區塊）和 index.html（第 06 項導航）。
